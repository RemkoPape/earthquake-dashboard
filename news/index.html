RSS Notes

Turn RSS feeds into clean Obsidian article notes.

This page describes the local clipping flow powered by OnlyNews.py. It skips AI processing, lets you define your own RSS sources, and keeps the notes connected to the wider dashboard through shared links and source pages.

Open script Back to hub Open documents Open earthquakes

How it works

1. Add feeds Edit the RSS_FEEDS block in OnlyNews.py and paste in the RSS URLs you want to track.
2. Run the collector The script reads the newest entries, skips anything already seen, and writes article notes plus source notes.
3. Review in Obsidian Each note uses the article template, includes the original link, and links back to Dashboard Hub, Documents, and Earthquakes. 
RSS_FEEDS = {
    "General": [
        "https://example.com/rss.xml",
        "https://example.org/feed",
    ],
    "Climate": [
        "https://example.net/climate.rss",
    ],
}

Output

Article notes New entries are written into news/articles/ by group, ready for Obsidian.
Source notes Feed-level notes are written into news/sources/ so the articles stay grouped by source.
Seen state The script tracks handled items in news/.onlynews-state.json to avoid duplicates.

Commands

python OnlyNews.py
python OnlyNews.py --dry-run
python OnlyNews.py --output "C:\Path\To\Your\Vault\News"

What gets created

The article notes follow your Obsidian template with title, aliases, date, source note, feed title, author, link, tags, summary block, key points, notes, article content, and related links. This page is just the public guide; the actual markdown notes are meant to be read and edited inside Obsidian.
